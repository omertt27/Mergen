Mergen Privacy Policy

Effective Date: May 14, 2026

TL;DR: Mergen runs 100% locally. Zero data leaves your machine. Period.

1. What Mergen Does

Mergen is a browser extension that captures console logs, network requests, and page context from your local development environment and sends them to a local server running on 127.0.0.1 (localhost).

2. Data Collection

Mergen collects:

  • Console logs — console.log(), console.error(), console.warn() output
  • Network requests — HTTP method, URL, status code, duration, request/response bodies
  • Page context — Current URL, page title, active element, localStorage, sessionStorage

3. Where Data Goes

All data stays on your computer.

  • ✅ Data is sent to http://127.0.0.1:3000-3010 (your local server only)
  • ✅ No external servers, no cloud services, no third parties
  • ✅ No analytics, no tracking, no telemetry
  • ✅ Your data never touches the internet

4. Data Storage

Mergen stores captured events in an in-memory ring buffer (200 events max) on your local server. When the server stops, all data is deleted.

The extension stores only:

  • Selected port number (default: 3000)
  • Mute state (per-tab)

5. Permissions Explained

  • storage — Save port preference and mute state
  • tabs — Identify which tab sent an event
  • alarms — Periodic health checks to local server
  • host_permissions — Send data to 127.0.0.1:3000-3010 only

6. Third-Party Access

None. Mergen does not connect to any external services. Your data is never shared, sold, or transmitted outside your computer.

7. Security

  • All traffic is to localhost only (127.0.0.1)
  • No HTTPS needed (no network exposure)
  • Optional shared-secret authentication between extension and server
  • PII redaction for sensitive data (tokens, passwords, emails)
