Welcome to Mergen

Live browser telemetry for your AI debugger — zero configuration.

1

Start the Mergen server

In your terminal, inside the server/ directory:

npm run build && npm start

The server listens on http://127.0.0.1:3000 by default. If port 3000 is busy it will try 3001–3010 automatically — check the terminal output.

2

Connect your MCP client

Add the Mergen server to your MCP client config (e.g. Claude Desktop, Cursor). Use stdio transport:

{
  "mergen": {
    "command": "node",
    "args": ["/path/to/mergen/server/dist/index.js"]
  }
}
3

Browse — Mergen records everything

Open any webpage. Console logs, errors, warnings, and network requests (fetch + XHR) are automatically captured and sent to the local server buffer.

4

Ask your AI debugger

In your MCP-connected client, ask something like:

What errors are happening in my browser right now?

Mergen will interpret the telemetry — errors first, then warnings, then a summary.

Check server connection

Make sure the server is running before you start debugging.

Click the Mergen icon in your toolbar anytime to see live stats, change the port, or pause recording on a tab.
