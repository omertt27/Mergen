Welcome to Mergen

Stream live browser telemetry into any AI IDE — everything stays on localhost.

1

Start the Mergen server

Quickest way — installs and auto-configures your IDE:

npx mergen-server@latest setup
Or, if you cloned the repo, run inside server/: 
npm run build && npm start

The server listens on http://127.0.0.1:3000 by default. If port 3000 is busy it will try 3001–3010 — check the terminal output.

2

Connect your AI IDE

Select your IDE for the exact setup command:

Run in your terminal from the Mergen repo root:

claude mcp add mergen --transport stdio -- node "$(pwd)/server/dist/index.js"

Verify: claude mcp list — you should see mergen listed.

Auto-setup (writes to .cursor/mcp.json or ~/.cursor/mcp.json):

npx mergen-server@latest setup
Or manually edit .cursor/mcp.json: 
{
  "mergen": {
    "command": "node",
    "args": ["/absolute/path/to/Mergen/server/dist/index.js"]
  }
}

Then: Settings → Tools → MCP — confirm "mergen" is listed.

Auto-setup (writes to ~/.codeium/windsurf/mcp_config.json):

npx mergen-server@latest setup
Or manually edit ~/.codeium/windsurf/mcp_config.json: 
{
  "mcpServers": {
    "mergen": {
      "command": "node",
      "args": ["/absolute/path/to/Mergen/server/dist/index.js"]
    }
  }
}

Restart Windsurf, then open Cascade → MCP Servers to confirm.

.vscode/mcp.json is already committed to the Mergen repo — no config needed if you opened the Mergen project in VS Code.

For other projects (global user-level config):

npx mergen-server@latest setup

Requirements: VS Code 1.99+, GitHub Copilot extension, Agent mode enabled. Open Copilot Chat → switch to Agent mode → click tools — Mergen tools should appear.

3

Browse — Mergen records everything

Open any webpage you're developing. Console logs, errors, warnings, and network requests (fetch + XHR) are automatically captured and forwarded to the local server buffer.

4

Ask your AI

In your MCP-connected IDE, ask something like:

What errors are happening in my browser right now?

Mergen surfaces errors first, then warnings, then a summary with suggested fixes.

Verify the full connection

Checks that the server is running and has received at least one event.

Click the Mergen icon in your toolbar anytime to see live stats, change the port, or pause recording on a tab.
