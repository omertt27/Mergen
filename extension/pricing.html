Common questions

What is analyze_runtime?

It's a premium MCP tool your AI assistant (Claude, Cursor, Copilot, etc.) can call to get a structured root-cause analysis of everything in the current buffer — console errors, stack traces, failed network requests — along with a proposed fix. Free tools give you the raw data; analyze_runtime gives you the interpretation.

Does Mergen send my logs to the cloud?

No. All telemetry stays on your machine. The only outbound calls are: license validation (key hash only), and LemonSqueezy usage records for overage/PAYG billing (just a count — no log content). Your actual log data never leaves localhost.

What happens when I hit my monthly credit limit?

On Solo Standard, additional calls cost $0.05 each and are not blocked — they're billed at month-end via LemonSqueezy usage-based billing. On the Free plan, analyze_runtime is fully blocked. Upgrade to Solo Pro or Team for unlimited calls.

How does overage billing work technically?

When you exceed your included credits, Mergen queues the count locally and reports it to LemonSqueezy's usage-records API in batches (every 5 seconds of activity). Pending counts survive server restarts and are flushed on next startup. You can always see the current overage count in the popup or at /usage.

How does Team sync work?

Each teammate runs the Mergen server locally. A shared token authenticates pushes. When teammate A's server captures events, it POSTs them to the built-in relay running on your machine (or an external relay you configure). Your MCP context then includes their events, so your AI can diagnose cross-machine issues.

The relay uses Server-Sent Events (SSE) for real-time streaming with automatic reconnection. No third-party infrastructure is required — the relay is built directly into the Mergen server.

Can I use one license on multiple machines?

Yes. LemonSqueezy license keys allow a configurable number of activations. Solo plans allow up to 3 machines. Team plans allow one activation per seat purchased.
