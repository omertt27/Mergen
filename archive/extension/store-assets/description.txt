Mergen — Local-First Runtime Debugging for AI

Stop copy-pasting console errors into your AI assistant. Mergen streams live browser telemetry directly to your AI IDE — completely local, no cloud required.

🎯 What Mergen Does:
• Captures console logs (log, warn, error)
• Tracks network requests (fetch, XMLHttpRequest)
• Records page context (URL, title, localStorage, sessionStorage)
• Sends everything to your local server (127.0.0.1)
• Your AI assistant reads it via Model Context Protocol (MCP)

✅ Privacy First:
• 100% local — all data stays on your machine
• No cloud services, no external servers
• No analytics, no tracking, no telemetry
• Zero data leaves localhost

🚀 Works With:
• Claude Code
• Cursor
• VS Code (GitHub Copilot)
• Windsurf
• Continue
• Any MCP-compatible AI IDE

💡 Perfect For:
• Debugging React/Vue/Angular apps
• Investigating API failures
• Understanding console errors
• Troubleshooting auth flows
• Finding CORS issues
• Analyzing network performance

🔧 How It Works:
1. Install this extension
2. Run: npx mergen-server setup
3. Ask your AI: "Why did that request fail?"
4. Your AI sees the console logs, network activity, and page context

🎓 Example Questions:
• "Get recent logs" — See latest console output
• "Why did that POST request fail?" — Network analysis
• "What's in localStorage?" — Page state inspection
• "Show me all 401 errors" — Auth debugging

📦 Setup (2 minutes):
npm install: npx mergen-server@latest setup
Documentation: https://github.com/omertt27/Mergen

🔐 Security:
• Localhost-only (127.0.0.1:3000-3010)
• Optional shared-secret auth
• PII redaction for sensitive data
• Open source (MIT license)
